Chapter 2

Diagnosis and Monitoring

This chapter turns the taxonomy into a monitoring contract. The main claim is that a serious LM run should separately observe attention entropy, logsumexp(logits), embedding-gradient norms, precision-sensitive paths, and decode-side behavior instead of collapsing everything into loss and global grad norm. Entropy Collapse PaLM GLM-130B OLMo

Why This Chapter Exists

The taxonomy only becomes useful once it is measurable. Many training stacks still log a single top-line loss, a global gradient norm, and perhaps a non-finite counter. That is enough to confirm that a run is bad; it is not enough to decide which chapter of this book to open first. The reason this chapter exists is to reduce the gap between “something went wrong” and “we know which control surface deserves the next debugging hour.”

The rerun bundle improved this chapter materially because the heavy-mode workflow forced a chapter packet and a failure playbook. That means the monitoring order below is not only a summary of beliefs. It is a structured response path extracted from the evidence.

Monitoring Contract

A minimally adequate monitoring stack for decoder-LM stability should cover four distinct locations plus one inference-side probe:

  1. Residual / backbone health. Log global grad norm and, if possible, a small set of layerwise norms to distinguish early geometry failure from later localized failure.
  2. Attention-score health. Log attention entropy per head or per layer, plus at least one scale statistic for attention logits. The Apple entropy-collapse paper is what justifies treating this as first-class rather than decorative instrumentation.
  3. Output-head health. Log logsumexp(logits) or equivalent log Z statistics if the LM head is suspected to drift. This is the monitoring partner of z-loss.
  4. Numeric-path health. Log non-finite fractions, mixed-precision overflow events, and any dtype-sensitive softmax or reduction path that can silently become brittle.
  5. Decode-side health. For repetition complaints, compare greedy against nucleus sampling before changing training. Holtzman makes this a diagnostic, not merely a generation preference.

Key Metrics And What They Mean

Attention entropy

This is the most important chapter-specific metric in the current evidence base. The Apple paper argues that late-stage instability is preceded by attention entropy collapse. That matters because it gives a signal that rises earlier and more specifically than waiting for a visible loss spike. Entropy Collapse

logsumexp(logits) or log Z

If the model’s output-head behavior is drifting while the trunk still looks comparatively ordinary, the global loss can lag the true problem. PaLM’s explicit z-loss = 10^-4 \log^2 Z is strong evidence that output-normalizer control is part of the training recipe, not a secondary afterthought. PaLM

Embedding-path gradient norm

GLM-130B reports that embedding-layer gradients could dominate early training instability and that shrinking them by 0.1 solved most numerical-instability problems in that stack. The practical lesson is not that every run needs embedding-gradient shrink. The lesson is that one layer family can carry most of the instability burden while global norms still look only moderately abnormal. GLM-130B

Softmax and reduction dtype

When infs or NaNs appear, the first question should not always be whether the architecture is wrong. In public reports, attention softmax dtype and FP32-sensitive optimizer or reduction paths matter materially. That is why numeric-path monitoring appears before any frontier optimizer discussion in this handbook.

Worked Diagnostic Path

Consider a run that trains normally for many thousands of steps, then shows a sudden loss spike, recovers partially after restart, and later produces repetitive generations. A weak diagnosis would treat all three observations as one failure. A stronger diagnosis proceeds in order. First ask whether any head-specific attention entropy collapsed before the spike. If yes, the attention-logit path is implicated and Q/K normalization or soft-capping becomes a coherent intervention family. If entropy looks ordinary, ask whether logsumexp(logits) rose steadily. If yes, the LM head deserves separate treatment via z-loss or, experimentally, OEC. If both are quiet and the run used a fragile FP16 path, inspect numeric-path details. Only after training-side monitoring is judged should the later repetition symptom be interpreted, starting with a greedy-versus-nucleus comparison.

This sequence sounds procedural because it is procedural. The main value of the monitoring chapter is not elegance but order.

Common Anti-Patterns

  • Watching only loss and global grad norm, then concluding the model “randomly exploded.”
  • Adding repetition penalties before verifying whether the issue disappears under nucleus sampling.
  • Treating all late spikes as optimizer problems even when attention entropy gave an earlier surface-specific warning.
  • Logging non-finite counts without also logging the dtype-sensitive operations where they are likely to originate.

Strengths And Limitations

Strength. The chapter gives a real intervention order and makes the rest of the handbook conditional on observable signals.

Limitation. The current public evidence still does not provide one shared dashboard trace containing all the recommended metrics on the same flagship run, so the chapter is best read as a well-supported forensic playbook rather than a reproduced monitoring benchmark.

Practical Notes

If a training platform is too constrained to log everything, log the metrics that change decisions. In the current evidence base, the most decision-relevant additions beyond loss are attention entropy, logsumexp(logits), one embedding-path gradient monitor, and a decode-side comparison. Those four additions do more to sharpen diagnosis than adding many generic per-layer summaries without a chapter-specific question behind them.
