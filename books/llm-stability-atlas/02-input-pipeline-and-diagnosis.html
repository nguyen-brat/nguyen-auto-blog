Chapter 2

Input Pipeline and Diagnosis

Stability is not only about architecture. The input pipeline determines the training targets the model sees, and the diagnosis loop determines whether you notice precursor signals before a run becomes expensive to recover. This chapter traces the standard decoder path from packed text blocks [B, T] into the residual stream [B, T, D], then turns that path into a concrete monitoring checklist. PaLM Small-Scale Proxies

Why This Chapter Exists

Engineers often jump directly from a spike to optimizer changes. That jump is risky because the apparent optimizer problem may actually be a data-packing issue, a boundary-token issue, or a metric visibility issue. The goal of this chapter is to show that diagnosis begins earlier: with how documents become token blocks and with which statistics you choose to watch during training.

Intuition

A clean causal-LM pipeline is simple in outline. Documents are tokenized into one long stream, explicit boundary markers such as [eod] are inserted, the stream is packed into fixed windows [B, T], embeddings turn those windows into [B, T, D], and the decoder preserves that backbone shape until the LM head yields logits [B, T, V]. The catch is that each of those steps creates different kinds of measurement opportunities and different ways to misread a failure.

Formal Mechanics

Let token IDs be input_ids ∈ ℤ^[B, T]. The embedding lookup gives

X₀ = E[input_ids] ∈ ℝ^[B, T, D].

Inside each decoder block, the hidden stream preserves the same outer shape while attention materializes per-head tensors Q, K, V ∈ ℝ^[B, H, T, d_h] and attention logits S ∈ ℝ^[B, H, T, T]. The output head finally produces L ∈ ℝ^[B, T, V]. If repetition-aware training is added, each position can also carry a set of negative candidates C_t, often represented as a padded [B, K_neg] structure.

Packed-example hygiene matters because the next-token targets depend on where document boundaries are placed. PaLM explicitly concatenates documents, inserts a special [eod] token, and then cuts exact 2048-token training sequences. PaLM

Worked Flow Trace

Suppose two short documents are tokenized into one stream:

[the, model, trains, smoothly, [eod], then, one, head, spikes, [eod]]

If the training length is T = 8, one batch element becomes:

input_ids[b] = [the, model, trains, smoothly, [eod], then, one, head]

labels[b] = [model, trains, smoothly, [eod], then, one, head, spikes]

After embedding lookup the model sees X₀ ∈ ℝ^[1, 8, D]. If generation later shows repeated [eod] or strange cross-document continuations, the right question is not only “which optimizer was used?” but also “what targets did packing create and how was end-of-document handled?”

Practical Diagnosis Set

Metric What it reveals Why it matters
Global grad norm Whether the overall update is exploding Necessary but too coarse to localize failure

Strengths

  • This pipeline view gives a concrete bridge from data, tensors, and metrics to chapter-specific interventions.
  • It prevents decode-time repetition and packing artifacts from being mistaken for optimizer failures.
  • It makes the diagnosis loop operational: the monitoring set is small enough to run in ordinary training code.

Limitations And Failure Cases

  • The same metric signature can still arise from different root causes, especially when multiple pathologies co-occur.
  • Public evidence is strongest for text-only dense decoders; omni-model balancing failures remain only narrowly covered here. [MISSING FROM SOURCE]
  • The chapter does not replace real run forensics such as checkpoint diffing or optimizer-state inspection.

Practical Notes

If you must choose only a few metrics, prioritize attention entropy, logsumexp(logits), and embedding-layer grad norm in addition to total loss. Those three are the most useful bridge between a vague symptom and a mechanism family. Also keep a recovery policy ready. PaLM’s public run history makes clear that even a strong preventive recipe benefits from checkpoint rollback and selective data-batch skipping when rare spikes still occur. PaLM Small-Scale Proxies

Evidence Notes

The small-scale-proxy literature is especially valuable here because it argues that some instability regimes can be detected from activation and gradient-scaling patterns before complete divergence. That is exactly the kind of evidence needed to make the metric set more than a logging superstition.
