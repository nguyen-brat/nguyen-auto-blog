Chapter 3

Stability Baseline Recipe

The practical baseline is not one paper copied verbatim. It is a stack synthesized from the strongest public anchors: Pre-LN or RMSNorm-based decoder blocks, warmup, global clipping 1.0, FP32-sensitive reductions, explicit [eod] packing, Q/K normalization or attention-logit soft-capping when needed, and z-loss for output-head control. This chapter states that recipe first, then explains why it is the right default before the more specialized chapters begin. PaLM OLMo 2 docs OLMo card

Why This Chapter Exists

A handbook that only criticizes methods but never commits to a working default is not useful in implementation practice. The point of this chapter is therefore pragmatic: if a research engineer must start or repair a dense decoder LM run today, what should the first stable recipe look like before frontier or specialized interventions are considered?

Intuition

The baseline recipe is built by assigning one control to each failure surface. Pre-LN or RMSNorm handles initialization-time gradient behavior. Q/K normalization or soft-capping constrains attention logits before softmax sharpness becomes pathological. z-loss constrains the LM head. Higher-precision reductions and optimizer states protect the numeric path. Explicit [eod] packing protects boundary behavior. The recipe is therefore modular rather than magical.

The Default Stack

Layer Default Public anchor
Backbone geometry Pre-LN or RMSNorm-based decoder Xiong 2020, OLMo family
Attention control Q/K normalization or soft-capping if attention logits drift OLMo 2 docs, Gemma 2 docs, entropy-collapse paper
Output head z-loss = 1e-4 · log² Z PaLM
Numeric path BF16 where possible, otherwise FP16 with FP32-sensitive ops PaLM, GLM-130B, OLMo
Grad control global grad clip = 1.0 PaLM, OLMo
Data packing Explicit [eod] insertion before exact-length packing PaLM
Monitoring Grad norm, embedding grad, attention entropy, logsumexp(logits) Synthesis across bundle sources

Concrete Reading Aid

The recipe becomes easier to remember if it is expressed as one sequence of choices:

  1. Use a Pre-LN or RMSNorm decoder baseline.
  2. Apply warmup and conservative early learning-rate growth.
  3. Keep FP32 where the numeric path is fragile: reductions, optimizer state, and attention softmax if FP16 is forced.
  4. Clip globally at 1.0, but do not mistake clipping for a complete solution.
  5. Add z-loss = 1e-4 · log² Z to protect the output head.
  6. Pack training data with explicit [eod].
  7. Add attention-logit controls if entropy or attention-score magnitudes drift upward.
  8. Only add repetition-specific losses after decode-time degeneration has been ruled out.

Strengths

  • The recipe is built from public large-scale precedents rather than from one-off heuristics.
  • It is modular: each component answers a distinct failure surface.
  • It stays implementable in ordinary modern PyTorch or Megatron-style stacks.

Limitations And Failure Cases

  • The recipe does not guarantee spike-free training. PaLM remains the strongest public counterexample.
  • It is still partly synthetic: no single public source publishes the full exact stack in one place.
  • Very deep stacks may still need DeepNorm or other residual rescaling beyond the baseline.

Practical Notes

Two exact numbers matter enough to be remembered directly: PaLM used exact-length 2048-token packed sequences, while the public OLMo 2 model card exposes a 4096 context configuration for the released 13B model. Likewise, the OLMo 7B public recipe exposes warmup 5000, peak LR 3e-4, epsilon 1e-5, weight decay 0.1, and global clip 1.0. Those are not universal constants, but they are strong evidence that the baseline should be anchored in concrete public precedent. PaLM OLMo card OLMo 2 card

Evidence Notes

The reason this recipe still includes a recovery playbook is that PaLM reported roughly 20 irregular spikes in the 540B run even after combining clipping and z-loss. The baseline is therefore “best public default,” not “final answer.”
